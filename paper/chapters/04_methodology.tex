\section{System Architecture}

The proposed framework is an intelligent Deep Reinforcement Learning system for security posture aware SFC placement. It consists of five principal components, as illustrated in Figure~\ref{fig:architecture}:

\begin{enumerate}
  \item \textbf{Substrate Network}: a dynamic graph whose node resources, link bandwidths, and security posture states evolve as VNFs are allocated, expire, and security incidents occur.
  \item \textbf{SFC Request Generator}: produces incoming SFC requests with random VNF demands and constraints.
  \item \textbf{SFC Placement Environment}: a Gymnasium-compatible MDP environment implementing the state, action, security posture-aware reward, and constraint logic.
  \item \textbf{Maskable PPO Agent}: the DRL backbone that trains on rollouts collected from the environment, updating its policy and value networks using the PPO objective.
  \item \textbf{GNN Feature Extractor}: encodes the substrate topology and node states — including current security posture — into rich embeddings for the DRL policy network.
\end{enumerate}

\section{DRL Backbone: Maskable PPO with GNN}

The learning algorithm is Maskable Proximal Policy Optimization (Maskable PPO), an action-masked extension of the PPO family of on-policy actor-critic methods. At each step the environment provides a binary mask over actions; the policy's logits are set to $-\infty$ for masked (infeasible) actions before the softmax, so the sampling distribution has zero support on infeasible placements and the value and policy gradients are computed only over feasible actions. This guarantees that the agent never selects a constraint-violating node during training or inference --- a stronger guarantee than the soft-penalty formulations used by prior DRL-based SFC placement methods, which only discourage violations through negative rewards~\cite{ran_dynamic_2023, zhang_dqn-msra_2025, taktak_drl_2024, xu_dynamic_2023, khemili_parallel_2024}.

The policy network uses a Graph Neural Network (GNN) as its feature extractor. The GNN reads the substrate graph topology and per-node features, propagates information between neighboring nodes through message passing, and produces a compact representation of the entire network state. Three GNN variants are supported --- GCN, GAT, and GraphSAGE --- allowing the policy to capture topology-dependent patterns (e.g., node centrality, neighborhood load, security posture clusters) that a flat MLP cannot represent. Previous DRL-based placement work has incorporated limited structural awareness through attention mechanisms~\cite{he_leveraging_2023} or parallelism-aware encoders~\cite{lin_lppif_2025, wang_fairness-aware_2025}, but the principled graph-message-passing approach adopted here yields embeddings that directly reflect the substrate graph. This graph-level representation is combined with the global context vector (Section~4.3.2) and fed into the actor-critic heads that output the placement decision and value estimate.

\section{Observation Space}

The environment exposes a \texttt{Dict} observation with four components. All values are normalized to $[0, 1]$ and padded to $N_{\max} = 25$.

\subsection{Node Feature Matrix}

The node feature matrix $\mathbf{F} \in \mathbb{R}^{N_{\max} \times 20}$ provides a rich per-node characterization of the substrate. For each real node $n \in \{0, \ldots, N-1\}$ the 20 features are organized into six groups, summarized in Table~\ref{tab:node_features}.

\begin{table}[!ht]
\centering
\caption{Node Feature Groups ($\mathbf{F} \in \mathbb{R}^{N_{\max} \times 20}$)}
\label{tab:node_features}
\begin{tabularx}{\textwidth}{c l X}
\hline
\textbf{Indices} & \textbf{Group} & \textbf{Features} \\ \hline
0--3  & Base resources      & Residual RAM, CPU, Storage (fraction of initial capacity); Effective security score $s_n^{\text{eff}} / s_{\max}$ \\
4     & Connectivity        & Average residual bandwidth to neighbors, normalized by $b_{\max}$ \\
5     & Distance            & Shortest-path latency from the previous placement node, normalized by $\ell_{\max}$; zero for the first VNF \\
6--8  & Global resource shares & $\text{used}_n^{r} / \sum_j \text{used}_j^{r}$ for $r \in \{\text{RAM, CPU, Storage}\}$: the fraction of all consumed resources sitting on this node \\
9     & VNF global share    & $\text{VNF}(n) / \sum_j \text{VNF}(j)$: fraction of all VNF instances hosted by this node \\
10    & SFC tenancy ratio   & $\text{SFC}(n) / \max(\text{VNF}(n), 1)$: chain diversity per instance, a blast-radius proxy \\
11--13 & VNF fit ratios     & $d_v^{r} / \tilde{r}_n^{r}$ for $r \in \{\text{RAM, CPU, Storage}\}$: how much of the node's remaining headroom the current VNF would consume \\
14    & Incident pressure   & Accumulated pressure $p_n$ \\
15    & Load norm           & $\text{VNF}(n) / \max(Q_{75}^{\text{VNF}}, 1)$: VNF count relative to the 75th-percentile reference \\
16    & Base incident prob.\ $q_n$ & As defined in Eq.~(\ref{eq:qn}): the same signal the risk formula and baselines use \\
17    & Expected incidents  & $q_n \cdot T_s / T_{\text{cap}}$, clipped to $[0, 1]$ \\
18    & Node risk score     & Equals $q_n$ (feature 16); an explicit risk slot \\
19    & Expected lost revenue & Inverse-normalized $1 - 1/(1 + \text{ELR}_n)$ where $\text{ELR}_n = P_n^{\text{hit}} \cdot \text{SFC}(n) \cdot R_{\text{ttl}} \cdot D_{\text{inc}}$ \\
\hline
\end{tabularx}
\end{table}

\textbf{Design rationale.} Features 0--5 capture the \emph{local} resource and topology state visible to a myopic heuristic. Features 6--10 add \emph{global} context: resource concentration, instance density, and blast-radius signals that help the agent detect ``hot-spot'' nodes — nodes that are operationally important because they host a disproportionate share of the workload. Features 11--13 bridge the current VNF demand to each candidate node, directly informing how tight a fit each placement would be. Features 14--19 expose the \emph{risk-driving} quantities — incident pressure, load, $q_n$, expected incidents, and expected lost revenue — that the risk model and baselines use internally, closing the observational asymmetry between the RL agent and the reward/risk formulas.

Padded nodes (indices $N$ through $N_{\max}-1$) are zeroed and excluded from GNN message passing via the node mask.

\subsection{Global Context Vector}

The global context vector $\mathbf{g} \in \mathbb{R}^{15}$ encodes the current SFC request and placement progress:

\begin{tabularx}{\textwidth}{c X}

0--2  & Current VNF demands: RAM / $d_{\max}^{\text{ram}}$, CPU / $d_{\max}^{\text{cpu}}$, Storage / $d_{\max}^{\text{stor}}$ \\

\hline
\end{tabularx}
\end{table}

The global context is processed by a two-layer MLP within the GNN feature extractor and fused with the graph-level substrate embedding before being passed to the actor-critic heads.

\subsection{Placement Mask and Node Mask}

\begin{itemize}
  \item \textbf{Placement mask} $\mathbf{m}^P \in \{0,1\}^{N_{\max}}$: 1 for nodes already used in the current partial placement, 0 otherwise. This is concatenated with $\mathbf{F}$ as the 21st input channel to the GNN, enabling the network to distinguish placed from unplaced nodes.
  \item \textbf{Node mask} $\mathbf{m}^N \in \{0,1\}^{N_{\max}}$: 1 for real substrate nodes, 0 for padding. The GNN processes only the real-node subset, avoiding meaningless computation on pad tokens.
\end{itemize}

\section{Action Space}

The action space is $\mathcal{A} = \{0, 1, \ldots, N_{\max} - 1\}$. At each step the agent selects a single substrate node index to host the current VNF. An action mask $\mathbf{m}_t^{\text{act}} \in \{0,1\}^{N_{\max}}$ restricts the agent to feasible nodes only; infeasible actions receive logits of $-10^8$ before the softmax, guaranteeing zero selection probability.

Each SFC request requires $K_r$ consecutive actions (one per VNF). Upon completion of the $K_r$-th action, the environment evaluates the full placement: if the end-to-end latency constraint holds, the SFC is accepted; otherwise, all allocated resources are released and the SFC is rejected.

\section{Action Masking Implementation}

The \texttt{action\_masks()} method of \texttt{SFCPlacementEnv} computes the binary feasibility mask at each step by:
\begin{enumerate}
  \item \textbf{Resource feasibility check:} A vectorized scan over all node resource matrices identifies nodes with $\tilde{\mathbf{r}}_n \geq \mathbf{d}_{v_k}$ and $s_n^{\text{eff}} \geq s_r^{\min}$.
  \item \textbf{Hard isolation check:} Hard-isolated nodes and (if $\delta_r = 1$) occupied nodes are excluded.
  \item \textbf{Bandwidth and latency budget check:} For each resource-feasible node, bandwidth $B(n_{k-1}, n) \geq b_r^{\min}$ and hop latency $L(n_{k-1}, n) \leq \ell_{\text{budget}}$ must hold, where the latency budget is:
  \begin{equation}
  \ell_{\text{budget}} = \max\!\left(0,\; \ell_r^{\max} - \hat{\ell} - (K_r - k - 1) \cdot \ell_{\min}\right),
  \end{equation}
  with $\ell_{\min} = 5$ ms (minimum link latency from config) and $\hat{\ell}$ the accumulated latency.
  \item \textbf{Safety fallback:} If no node is valid, action 0 is enabled so MaskablePPO can always sample; the environment then handles the forced rejection via \texttt{\_is\_valid\_action}.
\end{enumerate}

\section{Reward Design}

The reward function is structured to balance two objectives: maximizing SFC acceptance and minimizing security posture risk. It operates at two time scales: intermediate per-VNF step rewards and terminal per-SFC rewards.

\subsection{Intermediate Step Reward}

After placing VNF $v_k$ ($k < K_r$), the agent receives:
\begin{equation}
r_t^{\text{step}} = 0.1 - 0.3 \cdot \min\!\left(\frac{L(n_{k-1}, n_k)}{\ell_r^{\max}},\; 1\right).
\end{equation}
The constant $+0.1$ provides a progress bonus for each feasible VNF placement. The latency term penalizes distant placements, encouraging the agent to co-locate consecutive VNFs on nearby nodes to preserve latency budget for later hops.

\subsection{Terminal Acceptance Reward}

When all $K_r$ VNFs have been placed and the end-to-end latency constraint is satisfied, the environment computes the security cost heuristic $\rho_r$ (defined in Section~3.5.2), then returns:
\begin{equation}
r_T^{\text{acc}} = \left(R_{\text{acc}} - \lambda \cdot \rho_r\right) \cdot \left(1 + \omega \cdot \overline{\text{AR}}_w\right),
\end{equation}
where:
\begin{itemize}
  \item $R_{\text{acc}} = 1$ is the base acceptance reward;
  \item $\lambda = 4$ is the risk penalty coefficient, controlling the trade-off between acceptance and operational risk;
  \item $\rho_r \in [0,1]$ is the deterministic security cost heuristic for the placement;
  \item $\omega = 0.5$ is the windowed acceptance ratio (AR) modifier weight;
  \item $\overline{\text{AR}}_w$ is the acceptance ratio over the last $w = 10$ requests.
\end{itemize}
The AR modifier amplifies the reward when the agent has been accepting frequently, providing curriculum-style positive reinforcement.

\subsection{Terminal Rejection Reward}

When a placement fails (either by latency violation or forced rejection due to no feasible nodes), the agent receives:
\begin{equation}
r_T^{\text{rej}} = R_{\text{rej}} \cdot \left(1 + \omega \cdot (1 - \overline{\text{AR}}_w)\right),
\end{equation}
where $R_{\text{rej}} = -1$. The flipped AR modifier $(1 - \overline{\text{AR}}_w)$ amplifies the penalty when the agent has been rejecting frequently, providing stronger corrective signal during poor-performance phases.

\subsection{Revenue Tracking}

Upon acceptance, the environment records the request revenue $R_{\text{ttl}} \cdot \tau_r$ (where $R_{\text{ttl}} = 1.0$ is the revenue per TTL step). Rejected requests contribute zero revenue. This metric is not part of the reward itself but is tracked for evaluation alongside the expected lost revenue from incident downtimes.

\section{Best-Fit Baseline}

To assess the value of intelligent DRL-based security posture awareness over a greedy approach, a \textbf{Best-Fit by Minimum Risk} (\texttt{BestFitPlacement}) heuristic is implemented. For each VNF in the chain, it applies the same feasibility checks as the environment (including \texttt{apply\_incident\_view} for consistent constraint semantics, with downtime-blocked nodes having their resources zeroed), then selects the valid node with minimum per-node risk score, tie-breaking by resource waste (then node index):
\begin{equation}
n^* = \argmin_{n \in \mathcal{V}_t^{\text{valid}}} \left(\rho_n^{\text{node}},\; w_n^{\text{waste}},\; n\right),
\end{equation}
where the per-node expected-loss risk score is:
\begin{equation}
\rho_n^{\text{node}} = \min\!\left(q_n \cdot \text{SFC}(n) \cdot \frac{T_s}{\tau_{\max}},\; 1\right),
\end{equation}
with $q_n$ defined as in Eq.~(\ref{eq:qn}), $\text{SFC}(n)$ the number of active chains on node $n$, $T_s = \min(\tau_r, T_{\text{cap}})$, and $\tau_{\max}$ the maximum TTL. This is the per-node analog of the placement security cost heuristic (Eq.~\ref{eq:risk}). The waste tie-breaker is $w_n^{\text{waste}} = (r_n^{\text{ram}} - d_v^{\text{ram}}) + (r_n^{\text{cpu}} - d_v^{\text{cpu}}) + (r_n^{\text{stor}} - d_v^{\text{stor}})$.

This baseline is a strong comparator: it explicitly minimizes expected loss at each step, matching the primary objective of the DRL agent's security posture-aware risk penalty. It is used to assess whether the DRL agent's learned, intelligent policy provides any advantage over greedy risk minimization — in particular, whether the DRL agent exploits long-horizon reasoning about security posture dynamics that a myopic greedy strategy cannot.

\section{Training Procedure}

\begin{algorithm}[!ht]
\caption{Training Procedure for the Intelligent Security Posture Aware DRL Framework (Maskable PPO with GNN)}
\label{alg:training}
\begin{algorithmic}[1]
\Require Config $\mathcal{C}$, total timesteps $T_{\text{total}}$, rollout steps $n_{\text{steps}}$
\State Initialize substrate $G_S$ from $\mathcal{C}$, request generator $\mathcal{G}$ from $\mathcal{C}$
\State Create masked environment $\text{env} = \texttt{ActionMasker}(\texttt{SFCPlacementEnv}(\mathcal{C}))$
\State Build GNN feature extractor with \texttt{create\_edge\_getter(env)}
\State Initialize Maskable PPO agent $\pi_\theta$ with GNN policy
\State $t \leftarrow 0$
\While{$t < T_{\text{total}}$}
  \State \textbf{Collect rollout:} run $n_{\text{steps}}$ steps under $\pi_\theta$ with action masks
  \For{each step}
    \State $\mathbf{m}_t \leftarrow \text{env.action\_masks}()$
    \State $a_t \sim \pi_\theta(\cdot \mid \mathbf{s}_t, \mathbf{m}_t)$
    \State $\mathbf{s}_{t+1}, r_t, d_t \leftarrow \text{env.step}(a_t)$
  \EndFor
  \State \textbf{Compute GAE advantages} $\hat{A}_t$ using value function $V_\theta$
  \For{$n_{\text{epochs}}$ epochs}
    \For{each mini-batch of size $B$}
      \State Update $\theta$ using $L^{\text{CLIP}} + c_e H - c_v L^{\text{VF}}$
    \EndFor
  \EndFor
  \State $t \leftarrow t + n_{\text{steps}}$
  \If{$t \bmod f_{\text{eval}} = 0$}
    \State Run \texttt{run\_eval} and save best model if AR improved
  \EndIf
\EndWhile
\State \Return best checkpoint $\pi_\theta^*$
\end{algorithmic}
\end{algorithm}

During training, a \texttt{TrainingEvalCallback} periodically (every $f_{\text{eval}} = 20{,}000$ steps) calls the full evaluation pipeline on the training substrate to measure the agent's current acceptance ratio, risk metrics, and substrate utilization. The best model checkpoint (by acceptance ratio) is saved as \texttt{sfc\_ppo\_best.zip}.
