\subsection{Network Configuration}

The substrate network consists of 25 nodes connected as a complete graph ($p = 1.0$) with resources sampled uniformly as described in Section~3.1.3. Each node carries three independent CIA security scores (confidentiality, integrity, availability) sampled from $[1, 5]$, and is assigned to one of three trust zones (public, private, DMZ). Each link carries a security level sampled from $[0.2, 1.0]$. SFC requests consist of chains of 3 VNFs whose CIA minimum requirements and zone constraints are sampled as described in Section~3.2. The TTL follows a truncated exponential distribution on $[1, 25]$ with decay mean at the midpoint.

\subsection{Evaluation Protocol}

Evaluation uses 1000 SFC requests on the training substrate with a fresh request generator. To ensure fair comparison between the RL agent and the Best-Fit baseline, the Python and NumPy random number generator states are snapshotted before each algorithm's evaluation and reset to the same initial state, ensuring both algorithms process identical request sequences. The RL agent is evaluated in a single long episode (1000 requests, no resets), while the baseline uses a sustained-load simulation with the same substrate.

\section{Analysis and Discussion}

\subsection{Benefits of Action Masking}

Action masking provides three key benefits in this framework. First, it guarantees 100\% feasibility of accepted placements: since infeasible nodes are never selected, every placement that completes the chain satisfies all resource, bandwidth, latency, CIA security posture, trust-zone, and link-security constraints. Second, it dramatically improves sample efficiency: without masking, the DRL agent would waste many gradient steps learning to avoid infeasible actions through penalty signals. Third, it simplifies reward design: since constraint violations cannot occur, there is no need for complex penalty terms for individual constraint types.

\subsection{Effect of Multi-Dimensional Security-Aware Placement}

By modelling security as three independent CIA dimensions rather than a single scalar, the framework captures the practical reality that confidentiality, integrity, and availability requirements are independently specified and independently satisfied. The agent observes per-node CIA scores and per-SFC CIA minima in its observation space, enabling it to distinguish between, for example, a highly available but low-confidentiality node and a balanced high-security node — a distinction that a scalar security score would collapse. The windowed acceptance-ratio modifier provides additional stability: when the agent has recently been accepting many requests, the acceptance reward is amplified, encouraging continued good performance; when many rejections have occurred, the rejection penalty is amplified, pushing the agent to explore better strategies.

\subsection{Trust Zones and Link Security}

The trust zone mechanism allows operators to express placement affinity requirements beyond resource constraints. SFCs that require placement in a specific zone (e.g., private infrastructure for high-confidentiality processing) are both feasibility-filtered and zero-margined if misplaced — making zone compliance a first-class evaluation metric. The link security margin similarly captures path-level rather than node-level security: a VNF chain may traverse nodes with excellent CIA scores yet be connected by low-security links, and the link security margin makes this distinction explicit in the evaluation.

\subsection{GNN vs.\ MLP Policies}

The GNN-based feature extractor captures topology-dependent security and resource patterns that MLP-based policies cannot represent. For example, the DRL agent may learn to prefer substrate nodes that are central in the graph (with low average distance to all other nodes), reducing latency costs, or nodes with high residual resources, strong CIA scores, and the correct zone assignment. The zone one-hot encoding in the node feature vector (columns 16--18) allows the GNN to aggregate zone-specific neighbourhood information, enabling the policy to reason about whether a candidate node's zone-neighbours are compatible with the SFC's zone requirement across future VNF placements.

\subsection{Comparison with Best-Fit}

The Best-Fit baseline represents a strong greedy approach that enforces identical CIA, zone, and link-security hard constraints as the DRL agent and selects placement nodes by minimum resource waste, breaking ties by node ID. The key distinction is that the intelligent DRL agent optimizes security margins \textit{jointly with} acceptance ratio and long-term resource efficiency across the full SFC chain, while the baseline makes each VNF placement decision myopically. In heavily loaded scenarios, the DRL agent may choose nodes with a slightly smaller resource surplus early in the episode to preserve high-CIA, correctly-zoned nodes for future requests with stricter constraints — a look-ahead behaviour that a greedy step-wise baseline cannot exhibit.

\subsection{Scalability Considerations}

The current implementation uses a 25-node substrate, which enables efficient training within 100,000 timesteps. The GNN architecture is topology-agnostic by design: the \texttt{node\_mask} ensures that only real nodes are processed, and the edge index is reconstructed dynamically from the substrate graph. This means the same trained DRL policy can, in principle, be deployed on larger or reconfigured substrates while maintaining its multi-dimensional security awareness across different network configurations.
